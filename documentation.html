Template Documentation

This documentation explains the structure, JavaScript functionality, and CSS styling of your template. The code below is organized into sections, covering the header with navigation, main content area, call-to-action (CTA) buttons, FAQ accordion, optional content, and footer.

1. Template Structure Overview

The template is composed of the following key sections:

  • Header: Contains the logo, navigation (hamburger menu), and CTA buttons.
  • Main Content: A container for page-specific content.
  • Footer: Displays contact information, hours of operation, local info, and additional links.

5. FAQ Accordion

An FAQ accordion allows users to click on questions to reveal answers. The accordion’s behavior is managed via JavaScript. 

<div id="faq">
  <button class="accordion" aria-expanded="false">
    What services do we offer?
  </button>
  <div id="faq-panel-all" class="panel" aria-hidden="true">
    <p>Lorem Ipsum explains that our services include dental cleanings, checkups, and emergency dental care.</p>
  </div>
</div>

The JavaScript listens for clicks on the .accordion button and toggles the active class and inline styles on the panel.

6. Optional Content Section

The optional content block (#optional-content) is displayed only when it contains content. Otherwise, it is hidden. 

<div id="optional-content">

</div>

8. JavaScript Functionality

The JavaScript in your template adds interactivity by:

  • Loading external navigation from nav.html.
  • Toggling the hamburger menu and overlay.
  • Managing the FAQ accordion open/close behavior.
  • Conditionally displaying optional content.
  • Tracking non-CTA link clicks using Google Tag Manager.

For example, the accordion initialization script is as follows:

function initAccordion() {
  const accordion = document.querySelector(".accordion");
  const panel = document.getElementById("faq-panel-all");
  if (accordion && panel) {
    accordion.addEventListener("click", function () {
      this.classList.toggle("active");
      const isOpen = this.getAttribute("aria-expanded") === "true";
      this.setAttribute("aria-expanded", !isOpen);
      if (!isOpen) {
        panel.style.maxHeight = panel.scrollHeight + "px";
        panel.setAttribute("aria-hidden", "false");
      } else {
        panel.style.maxHeight = null;
        panel.setAttribute("aria-hidden", "true");
      }
    });
  }
}

9. CSS Styling Overview

The CSS file begins with a reset and base styles to remove browser inconsistencies. It then defines:

  • Typography & Headings: Styles for h1, h2, h3, paragraphs, and links.
  • Header & Navigation: Layout for the header, logo, hamburger menu, mobile menu, and overlay.
  • CTA Styles: Button appearance and hover effects.
  • FAQ Accordion: Styling for accordion buttons and panels with smooth transitions.
  • Optional Content: A bordered and shadowed container that pops out.
  • Responsive Behavior: Media queries adjust typography and layout for mobile devices.
  • Footer Structure: Multi-column layout, link styling, and background colors.

To change the look of a button, for example, you might update the CSS like so:

.cta {
  background: #1e3a5f;  /* Default button color */
  padding: 16px 24px;
  border-radius: 8px;
}
.cta:hover {
  background: #28b293;  /* Hover state */
}
